v0.15.11 - May 2026 — Engine v2.7
- RFWN width loosened back to fit "what's next" on a single line. v0.15.9's tightening (font × 6.2, cap 80) was too aggressive — on 970×250 at font 16, "what's next" overflowed and the tagline wrapped to 3 lines. New: font × 6.8 with cap 100, so the longer of the two natural lines fits as a single row while keeping the box snug.
- Wide-banner heading (h ≤ 100, aspect > 2) now vertically centres in the canvas instead of hugging the top. The side-by-side heading + subhead pair reads as a centred block rather than upper-aligned. 728×90 heading y goes from ~9 to ~18; subhead vertically centres on heading as before, so the pair sits at canvas mid-y.
- New "Live linking" section in the Auto-Resize Settings modal. Master toggle enables / disables real-time propagation for auto-resized elements. When on, every target element joins the source's link group with `group.liveLink = true` and edits on the source update every target instantly. When off, target elements are independent copies. Five property toggles inside (Text content / Font family + weight / Colour / Opacity / Animations + effects) drive which properties actually sync. Position, size, and font size are always independent per canvas — that's the point of the resize, so they're not toggleable.
- Sub-toggles dim and become non-interactive when the master is off so the hierarchy is visually obvious.
- Engine bumped to v2.7 to reflect the placement tweaks + the live-linking control surface.

v0.15.9 - May 2026 — Engine v2.5
- Logo + RFWN shrink 25% on tall formats (h > w, i.e. 300×600 and 160×600). They share the safezone top row in tall layouts and the previous sizes packed too tightly against each other. Logo height multiplied by 0.75 (32 → 24 for 300×600, 23 → 17 for 160×600). RFWN font multiplied by 0.8 (14 → 11 for 300×600, 10 → 8 for 160×600). Wide banners and square canvases unchanged.

Heading boxes have a generous 4–5 line wrap budget; the trailing padding (when the text wraps to fewer lines) was creating a visible gap between heading text and subheading text. The overlap eats that padding without colliding with the actual heading lines.
- RFWN bounding box hugs the wrapped text tighter. Width formula changed from `fontSize × 7` to `fontSize × 6.2` with a tighter max clamp (90 → 80). "what's next" (11 chars, the longer of the two natural wrap lines) is the new sizing target so the box right edge sits flush against the text right edge instead of leaving trailing slack.

- CRICOS font now takes the max of two formulas: minDim × 0.023 (original) and width × 0.008 (new). Bumps 728×90 from 4 → 6 and 970×250 from 6 → 7 where there's plenty of horizontal room. Other sizes unchanged.

v0.15.6 - May 2026
- Auto-Resize role detection now refreshes auto-assigned roles every sweep. Previously, once an element was auto-classified its role was frozen — improvements to the detector (like the aspect-ratio logo heuristic) wouldn't take effect on existing projects without resetting each layer by hand. The refresh only applies to auto-assigned roles; manually-set roles are still preserved.
- New "Show canvas selection dialogue" setting in the Auto-Resize Settings panel (gear icon). On (default): clicking Auto-resize opens the run modal where you pick targets, configure unassigned-handling, and review the destructive-clear warning. Off: clicking Auto-resize skips the modal entirely and runs the engine on every other canvas immediately. Combined with the technical progress overlay also off, the resize happens fully instantly with no intermediate UI at all — useful when you're iterating quickly and the target set never changes.
- New "Include unassigned elements by default" setting controls the value used by the bypass path AND pre-fills the run modal's checkbox. The run modal also remembers whatever you ticked last so it carries over between sessions.

v0.15.5 - May 2026
- RMIT logo: always top-right of safezone now. The previous bot-left mode for ultra-narrow skyscrapers (aspect < 0.4) packed the logo against RFWN + CTA in a cramped 30-px-tall bottom strip on 160×600. Top-right is consistent across every format and pairs naturally with RFWN top-left.
- RFWN: top-left mode extended to cover the skyscraper case too (aspect ≤ 2.0). Was previously bot-right for skyscrapers, which is what caused the bottom-pile-up. The bot-right mode is now reserved for wide banners only (aspect > 2.0). Text always justifies toward the closest canvas edge — left for left-anchored, right for right-anchored — never centered. The R1 logo↔RFWN snap is updated accordingly: ≤2 shares top edge, >2 shares right edge.
- Subheading font now scales from canvas dimensions rather than as a fixed % of heading font. New formula clamp(7-10, h × 0.16, 26) on banners and clamp(12, √area × 0.05, 26) elsewhere — tuned within ±2 of the user's reference data on every canvas. Fixes the "sub headline too small" issue on 728×90 / 970×250 / 320×50.
- No-drop policy: every placement rule now returns geometry rather than null. Subheading without a heading-anchor parks at safezone top-left. Main-image without a usable slot falls back to a centred minimum-size box (with a 24-px floor). Extra-info without any candidate slot falls back to a small box at safezone bot-left. Drop behaviour is now reserved exclusively for role='misc' when the user's "include unassigned" toggle is off.

v0.15.4 - May 2026
- Auto-Resize tuning batch from a second real-canvas test (300×250 → all sizes):
- Heading on tall / stacked canvases (300×600, 160×600) is no longer crushed to a one-word-per-line column. The CTA-clearance constraint was firing for tall-mode CTAs too — but tall-mode CTAs sit BELOW the heading, not to the right, so there's no horizontal conflict. The constraint now only applies when canvas aspect > 2 (i.e. CTA is truly on the right side). The logo constraint also now requires logo.y to actually overlap the heading band.
- Heading on tight wide banners (728×90, 320×50) gets a roomier column — width bumped from canvas.w × 0.32 to × 0.42 so "It's not too late to study in 2026." fits across 2 lines without cropping. Mid-banner heights (h ≤ 300, aspect > 2) bumped from 0.36 to 0.40.
- maxFontSize is now set alongside fontSize on every text-bearing rule (heading, subheading, cricos, rfwn). Source elements that had `autoSize: true` + `maxFontSize: 68` no longer blow past the per-canvas font cap — the auto-fitter caps at whatever the rule computed.
- Logo detection: small horizontal images (aspect ≥ 2.0, area < 18% of canvas) are now classified as `rmit-logo` regardless of the Always-Top persistence flag. Previously a logo dropped into the Main Layers section would fall through to `main-image`, get placed by the slot-search rule, and disappear from the brand corner on every target.
- Mask system survives auto-resize: shapes with `isMask: true` now preserve the mask flag through cloning, and a new post-pass remaps each mask's `maskTargetId` from the source image id to the cloned target image id, plus aligns the mask's x/y/w/h to the target image's new geometry. If the target image didn't transfer (e.g. main-image rule disabled in settings), the mask flag is removed so the shape renders as a normal shape rather than silently covering empty space.

Everything that powers the new rule-based auto-resize feature lives there now — role taxonomy, role detection / sweeps, all 9 placement rules, cross-role relations, post-placement passes (collision resolution + canvas clamp), the main executor, settings storage + modal, the run modal, the role-picker dropdown, the fake progress overlay, and the two button click listeners. v0.15.2 - May 2026
- Auto-Resize tuning, four changes informed by first real-world test (300×250 → all sizes):
- Heading: tight horizontal banners (h ≤ 100) get a smaller font cap (max 22, was 68) and slightly narrower column (canvas.w × 0.32 instead of 0.36) so the headline doesn't crop or overrun the image. Heading width is now also constrained by already-placed CTA + logo positions, so it can't bleed under them. Height is computed from the chosen font + a 2-line wrap budget, clipped to remaining canvas height — no more 82px-tall heading boxes on a 90px canvas.
- Subheading: on tight horizontal banners (h ≤ 100), subheading now sits to the RIGHT of the heading instead of below — vertically centred on the heading box. There isn't enough room to stack them on a 90px-tall canvas, so side-by-side it is. Subheading font is recomputed as ~60% of heading font.
- RFWN justification: text now justifies to match its placement mode — left-justified when RFWN sits at the top-left, right-justified when it sits at the bot-right. Never centred.
- Collision pass: post-placement, the five "never-touch" roles (logo, CTA, heading, subheading, RFWN) get walked pairwise in priority order. If any two overlap, the lower-priority one shrinks along whichever axis the centres are most offset on, with a 4-px clearance gap. Higher-priority element never moves.
- Canvas-clamp pass: every role except main-image and background-image is now forced fully inside the canvas. Off-canvas portions get clipped by adjusting x/y/width/height. Main image and bg are exempt because cover-mode placements deliberately overflow.

v0.12.0 - May 2026
- New layer-based masking system. Right-click a shape layer (rectangle, circle, or pixel — not line) on a non-persistent frame and pick "Use as mask" to turn it into a mask for the image directly beneath it. The shape's outline and stroke disappear on the canvas; the image below is clipped to the shape, with the mask carrying its own independent animation.
- Mask layers show a solid eye icon in the Layers panel (white when active, grey when hidden) so they're easy to tell apart from regular layers. Hiding the mask layer turns the mask off and the image reverts to its full unmasked form.
- Mask layers are mutually exclusive with link groups and dynamic data. When a mask is selected, both the Dynamic Data and Link Group panels show a clear "disabled while this layer is a mask" notice; the right-click → Link Group submenu is suppressed too.
- Persistent (Top/Bottom) layers cannot host masks. Dragging a mask into a persistent layer slot drops its mask flag automatically; the "Use as mask" entry is hidden when the shape is already pinned to a persistent layer.
- Saving a masked shape to the Assets library strips the mask flag so the asset comes back in as a plain shape — useful for treating mask shapes as reusable graphics without dragging the masking relationship across projects.
- Export pipeline (HTML5 / preview) emits the same SVG-mask construction so masked images export pixel-for-pixel the same way they look in the editor.

- Signing out now flushes the local autosave and reloads back to the splash + sign-in gate instead of leaving the app open in a half-signed-out state.
- Pushing to the cloud now checks for a same-name collision in the current context. If another project shares the name, a warning toast appears with "Replace" (overwrite the existing cloud copy) and "Rename" (push as a new project under a different name) — pushes with unique names go through silently as before.
- Lowered the minimum supported viewport from 1920 × 1080 back to 1366 × 768 — closer to what most laptops can give without external displays.

v0.10.0 - May 2026
- Loading screen now doubles as the sign-in gate: tabbed Sign in / Sign up form, "Remember me on this device" checkbox, and a "Use locally without signing in" escape hatch for users who'd rather skip the cloud. The form takes over the splash's quip + progress bar; the logo + glow stay behind it. Remembered sessions skip the gate entirely and the splash auto-dismisses as before.
- Added team Spaces — multi-workspace collaboration with a switcher in the chip dropdown (Personal + each space you belong to). The current space's name appears next to your email in the top bar.
- Invitations via shareable join link: from Manage Spaces → Invite, generate a one-time link that's copied to your clipboard. Recipients land at /?invite=… and on sign-in are auto-joined to the space (if the email matches).
- Cloud Projects panel now scopes to the current context (Personal or active Space) and shows the space's folder tree on the left. Create folders with "+ New folder", assign projects to folders inline, delete folders on hover. Pushes from the current context save into the active space (and selected folder if any).
- New SQL required in Supabase: spaces, space_members, space_invitations, folders tables + updated RLS policies on projects and storage. See the GitHub Issue / release notes for the SQL block.

v0.9.0 - May 2026
- Optional account sign-up and log-in (email + password) via a top-bar chip. Anonymous local use is unchanged — sign-in only unlocks cloud features and never blocks the app.
- New "My Cloud Projects" panel (accessible from the chip dropdown or the File menu) for pushing the current project to Supabase storage and pulling any of your cloud-saved projects back into the workspace. Pushed projects use the same .flow format as local saves, so nothing has to be re-imported.
- Added a stable per-project ID (`state.projectId`) so cloud pushes update the same record rather than creating duplicates. Existing local projects get one assigned on first open.

v0.6.0 - May 2026
- Data & Versions (dynamic creative): bind named element “slots” to spreadsheet columns and generate one finished ad set per row — ideal for spinning up the same banner set across many RMIT courses. Open it from File → Data & Versions or the Data button in the top bar.
- Per-element dynamic opt-in: a new “Dynamic Data” section in the Properties panel lets you mark exactly which fields vary per version (text & colour on text, + background on buttons, image on images, fill colour on shapes). Toggles propagate across a link group, so one logical slot stays consistent on every size.
- Composable with link groups: a slot maps to its link group when one exists (one binding fans across all sizes) or to a single element otherwise — without ever altering your link-group sync settings.
- Version switcher in the top bar applies the selected row live in both editing and preview, non-destructively — your template defaults are never overwritten.
- Edit-in-place: changing a dynamic slot on the canvas while a version is active writes back to that row’s cell. A new Data lock button makes dynamic slots read-only so you can review versions without nudging the data.
- ClickTag is bindable per version, and “Export All Versions” produces one folder per row (named from your chosen key column) through the standard Google-Ads export pipeline. The data sheet is stored inside the .cook project (auto-saves & travels) and can be imported/exported as CSV.

v0.5.1 - May 2026

v0.5.0 - May 2026
- Auto-resize from selected (AI): build your entire size set in one click. It reads every element on the selected canvas, detects each one’s role (heading, subheading, button, logo, shape, background image, or generic), then clears the other canvases and re-places + re-sizes matching elements using per-format layout presets.
- Auto-resize automatically links every propagated element into its own group with role-aware sync defaults — content and appearance stay in sync across canvases while position, dimensions and font-size remain independent per format.
- Added a dedicated "Font size" sync property for text link groups, split out from "Font settings" — you can now sync the typeface across canvases while keeping per-canvas sizes.
- Added seamless local auto-save: projects are continuously persisted to the browser (IndexedDB) and restored on reload, with a live save-status indicator (All changes saved / Saving… / Unsaved) in the top bar. Manual .cook saving is unchanged.
- New Project wizard now lets you pick which canvas sizes to include, the project name, the default canvas background colour, and a configurable maximum ad weight (KB) that drives the live size-validation warnings.
- Cleaned up the Tools panel — removed the permanent highlight on the Auto-resize and Toggle Safezones buttons (the AI badge stays).
